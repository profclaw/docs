## Security Architecture

profClaw is designed with a defense-in-depth model. Security is enforced at multiple independent layers - a failure in one layer does not compromise the system.

```
User Input
    |
    v
[Prompt Guard]          - Detect injection, jailbreak, token smuggling
    |
    v
[Security Mode]         - deny / sandbox / allowlist / ask / full
    |
    v
[Tool Registry]         - Tier filtering, availability checks
    |
    v
[Schema Validation]     - Zod parse, type enforcement
    |
    v
[FsGuard / SsrfGuard]  - Path and URL validation per operation
    |
    v
[Execution]             - Docker sandbox or local with policy
    |
    v
[Audit Logger]          - Immutable event log
```

## Risk Levels

All security events are classified by risk level:

| Level | Score | Meaning |
|-------|-------|---------|
| `LOW` | 0-24 | Normal operation |
| `MEDIUM` | 25-49 | Worth noting, not blocked |
| `HIGH` | 50-74 | Reviewed, may be blocked depending on mode |
| `CRITICAL` | 75-100 | Blocked and alerted |

## Default Security Configuration

Out of the box, profClaw runs in `standard` mode:

```yaml
security:
  mode: standard          # standard is safe for most deployments
  fsGuard:
    enabled: true
    allowedPaths:
      - "{{ workdir }}"   # project directory
      - "{{ tmpdir }}"    # system temp
  ssrfGuard:
    enabled: true
    allowedHosts: []      # no internal hosts allowed by default
  promptGuard:
    enabled: true
    blockThreshold: 25    # block inputs scoring >= 25
    warnThreshold: 10
  auditLog:
    enabled: true
    retention: 90         # days
```

## Security Responsibilities

| Layer | Your Responsibility | profClaw's Default |
|-------|--------------------|--------------------|
| Network | Set allowed external hosts | Block all private IPs |
| Filesystem | Define allowed paths | Block secrets, system files |
| Authentication | Configure your auth provider | Verify tokens per request |
| Secrets | Store in env vars, not config | Never log or expose |
| Models | Choose trusted providers | Validate responses |
