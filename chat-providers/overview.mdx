## Multi-Account Support

profClaw supports multiple accounts per provider:

```yaml
# settings.yml
chat:
  slack:
    accounts:
      - id: work
        bot_token: xoxb-work-...
        is_default: true
      - id: community
        bot_token: xoxb-community-...
```

## Architecture

All providers implement the same interface:

- **Inbound**: Receive messages via webhooks or polling
- **Outbound**: Send messages and responses
- **Status**: Health checks and connection monitoring
- **Auth**: OAuth flows or manual token setup

```
User Message
    |
    v
[Chat Provider]  ----incoming----> [Message Handler]
                                         |
                                    [AI Executor]
                                         |
[Chat Provider]  <---outbound-----  [Response]
```

## Webhook Setup

Most providers require a public webhook URL. Use `profclaw serve` to start the server, then expose it:

```bash
# Development: use ngrok
ngrok http 3000
# Then use https://your-ngrok-id.ngrok.io as your webhook base URL

# Production: configure your domain
WEBHOOK_BASE_URL=https://profclaw.yourdomain.com
```

## Configuration Pattern

All providers follow the same environment variable pattern:

```bash
# Enable a provider by setting its credentials
SLACK_BOT_TOKEN=xoxb-...
SLACK_SIGNING_SECRET=...

DISCORD_BOT_TOKEN=...

TELEGRAM_BOT_TOKEN=...
```

## Health Check

```bash
profclaw doctor --chat
```

This shows connection status for all configured chat providers.
