## Auto-Selection

When multiple providers are configured, profClaw picks the best available one automatically using a priority order. Cloud providers with tool-calling support are preferred over local models.

```bash
# profClaw selects from this order (first configured wins):
# anthropic -> openai -> azure -> google -> groq -> xai -> mistral ...
# -> ollama (always last)
```

You can override the default at any time:

```bash
profclaw config set provider anthropic
```

## Configuration

<Tabs>
  <Tab title="Environment Variables">
    Set API keys directly in your environment or `.env` file:

    ```bash
    # Recommended: set the providers you use
    ANTHROPIC_API_KEY=sk-ant-...
    OPENAI_API_KEY=sk-...
    GOOGLE_API_KEY=AIza...
    GROQ_API_KEY=gsk_...
    ```

    profClaw reads these at startup and configures each provider automatically.
  </Tab>
  <Tab title="settings.yml">
    ```yaml
    providers:
      default: anthropic
      anthropic:
        api_key: "${ANTHROPIC_API_KEY}"
      openai:
        api_key: "${OPENAI_API_KEY}"
      ollama:
        base_url: "http://localhost:11434"
    ```
  </Tab>
  <Tab title="CLI">
    ```bash
    profclaw config provider add anthropic --key sk-ant-...
    profclaw config provider list
    profclaw config provider set-default openai
    ```
  </Tab>
</Tabs>

## Using Model Aliases

Model aliases let you reference models by short names instead of full IDs:

```bash
# These are equivalent:
profclaw chat --model opus
profclaw chat --model claude-opus-4-6

# Provider/model shorthand also works:
profclaw chat --model anthropic/claude-opus-4-6
```

Common aliases:

| Alias | Provider | Model |
|-------|----------|-------|
| `opus` | Anthropic | claude-opus-4-6 |
| `sonnet` | Anthropic | claude-sonnet-4-5 |
| `haiku` | Anthropic | claude-haiku-4-5 |
| `gpt` | OpenAI | gpt-4o |
| `gemini` | Google | gemini-1.5-pro |
| `groq` | Groq | llama-3.3-70b-versatile |
| `local` | Ollama | llama3.2 |
| `grok` | xAI | grok-2 |
| `mistral` | Mistral | mistral-large-latest |
| `deepseek` | DeepSeek | deepseek-chat |

## Resilience

All providers include automatic retry with exponential backoff for transient errors (429, 503, network timeouts):

```bash
AI_MAX_RETRIES=2          # Default: 2 retries
AI_PROVIDER_TIMEOUT_MS=120000   # Default: 120 seconds
```

## Health Check

```bash
profclaw doctor --providers
```

This checks connectivity for all configured providers and reports latency.
